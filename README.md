# 🧟 ZOMBIE: PROJECT ZERO (Third-Person Zombie Survival)

> **Unity 엔진을 활용하여 구현한 3인칭 탑다운 뷰(Top-Down View) 좀비 서바이벌 액션 게임입니다. 로우 폴리(Low-Poly) 스타일의 환경 배경을 바탕으로, 데이터 주도형 무기 시스템, 유한 상태 머신(FSM) AI, 그리고 UI 기반 데이터 영속성(Ranking System)을 아키텍처 단위에서 직접 설계하고 최적화한 프로젝트입니다.**

## 📌 핵심 구현 명세 (Key Mechanics)

### 1️⃣ 인트로 씬(Intro Scene) 및 유저 세션 관리
* **플레이어 네이밍 시스템:** 랭킹 등록을 위해 인트로 씬에서 `TMP_InputField`를 통한 사용자 이름 입력 폼 구축
* **씬 전환 제어:** `SceneManager.LoadSceneAsync`를 이용해 인트로 비동기 로딩 및 부드러운 인게임 메인 씬 진입 프로세스 제어

### 2️⃣ 절대 좌표계 기반 탑다운 이동 및 마우스 트래킹 회전 물리
* **절대적 상하좌우 이동 (W, A, S, D):** 카메라 및 캐릭터 기준의 상대적 이동이 아닌, 월드 좌표계 기준의 직관적이고 고정된 절대 상하좌우 물리 역학 구현
* **레이캐스트 기반 시선 회저:** 마우스 커서의 스크린 좌표를 `Camera.main.ScreenPointToRay`를 통해 3D 월드 공간의 지표면 평면과 충돌(`Physics.Raycast`)시켜, 플레이어가 실시간으로 마우스 조준 방향을 정확히 바라보도록 회전 제어 (`Quaternion.LookRotation`)

### 3️⃣ 플레이어 상태 아키텍처 & 가시성 UI
* **스탯 제어 시스템:** Max HP 100 기준의 플레이어 생존 스탯 알고리즘 수립
* **가시성 스크린 UI:** 캐릭터 주위에 스크립트 기반으로 실시간 동기화되는 **빨간색 원형 체력 바(HP Circle UI)**를 배치하여, 인게임 전투 중 직관적인 스탯 피드백 제공

### 4️⃣ 아이템 스포너 기반 4종 무기(Weapon) 모듈 인터페이스
* **기본 무기 (우지 - Uzi):** 데미지 25, 발사 간격 0.12초, 탄창 용량 25발의 히트스캔 연사 로직 구현
* **동적 획득 및 10초 타임 오버 플로우 정책:** 아이템 스포너에서 무기 습득 시 특수 무기로 즉시 스왑되며 10초간 가동, 제한 시간 만료 시 기본 무기로 복귀하며 탄창이 Max로 충전되는 코루틴(`Coroutine`) 제어 구조 확립
* **특수 무기 및 발사 가시성 확장:**
  * **AK74 (강화형):** 데미지 50, 탄창 용량 45발의 고화력 단발 직진도 향상
  * **RPG7 (폭발형):** 데미지 500 세팅을 통한 범위 내 즉사급 포물선 스플래시 대미지 연산
  * **M4 (범위형):** 단일 발사 시 5발의 총알 프리팹이 방사형 부채꼴 각도로 확산 연사되는 산탄 메커니즘 전개
  * **Smoke Grenade (유틸리티):** 획득 즉시 5초간 대미지 처리를 무시하는 무적(`Invincible`) 상태 머신 가동

### 5️⃣ 가변 스펙 기반 3종 좀비 AI 시스템
* `NavMeshAgent` 추적 알고리즘을 확장하여 개별 스펙에 맞는 인공지능 속도 및 감지 레이아웃 최적화
  * **기본 좀비:** HP 100 / 공격력 20 / 이동 속도 2.0 (안정적인 팩 추적)
  * **뛰는 좀비:** HP 50 / 공격력 10 / 이동 속도 4.0 (돌격형 러시 패턴)
  * **강화 좀비:** HP 250 / 공격력 40 / 이동 속도 1.5 (탱커형 묵직한 압박)

### 6️⃣ UI 데이터 영속성 랭킹 시스템 (Ranking System)
* **점수 스코어링 공식:** 좀비 처치 점수 및 코인 획득 점수에 '살아남은 시간(Survival Time)' 변수를 실시간 가산 연산하는 스코어 컨텍스트 구조 설계
* **UI 데이터 바인딩:** 획득한 총 스코어와 인트로에서 입력한 유저 네임을 결합하여 실시간으로 순위를 정렬하고 표출하는 인게임 랭킹 스크롤 UI 아키텍처 마감
